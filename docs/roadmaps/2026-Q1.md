### Testing

**2026-02-18** — Added unit tests for all internal packages. Promoted
`github.com/stretchr/testify` from indirect to direct dependency.

| Package | File | Coverage |
|---|---|---|
| `internal/certs` | `format_test.go`, `parser_test.go` | 100% |
| `internal/prefs` | `prefs_test.go` | 82.6% |
| `internal/resources` | `fetcher_test.go`, `localroots_test.go`, `helpers_test.go` | 83.3% |

Key techniques used:
- Table-driven tests with `testify/assert` and `testify/require` throughout
- Dynamic test certificate generation via `crypto/ecdsa` + `x509.CreateCertificate`
  (no static PEM fixtures)
- Filesystem isolation using `t.Setenv("XDG_CACHE_HOME", t.TempDir())` and
  `t.Setenv("XDG_CONFIG_HOME", ...)` — avoids touching the real user config
- `httptest.NewServer` for `EnsureCCADBCSV` network tests
- `t.Skip` guard for the Linux-only `EnsureLocalRootsJSON` integration test

## Phase 2 — Platform Support & Distribution (partial)

### Windows Trust Store

**2026-02-18** — Implemented `collectRoots()` for Windows in
`internal/resources/localroots_windows.go` using `golang.org/x/sys/windows`:
- Opens the system `ROOT` certificate store via `windows.CertOpenSystemStore`
- Enumerates entries with `windows.CertEnumCertificatesInStore`
- Copies DER bytes from Windows-owned memory before context is freed
- Feeds into the shared `EnsureLocalRootsJSON` / `local_roots.json` / `LoadLocalRootsSKISet` pipeline
- "Compare Local vs CCADB" works end-to-end on Windows

### macOS Trust Store

**2026-02-18** — Implemented `collectRoots()` for macOS in
`internal/resources/localroots_darwin.go` using the `security` CLI:
- Runs `security find-certificate -a -p /System/Library/Keychains/SystemRootCertificates.keychain`
- Parses PEM output with the same `pem.Decode` loop used on Linux
- Supports context cancellation via `exec.CommandContext`
- Updated `localroots_unsupported.go` build tag to `!linux && !windows && !darwin`
- Added `localroots_darwin_test.go` integration test (skips when `security` not in PATH)
- "Compare Local vs CCADB" works end-to-end on macOS

### GitHub Actions CI / Release Pipeline

**2026-02-18** — Added `.github/workflows/`:
- `ci.yml` — runs `go test ./...` on push/PR for Linux
- `release.yml` — builds release artifacts on tag push:
  - `cert_viewer-linux-amd64`
  - `cert_viewer-windows-amd64.exe` (cross-compile via `GOOS=windows`)
  - `cert_viewer-darwin-amd64` and `cert_viewer-darwin-arm64`
- `README.md` updated with build status badge

## Phase 3 — Core Feature Gaps (partial)

### Preferences: Editable CCADB URL and Refresh Interval

**2026-02-18** — Extended `ShowPreferences` in `internal/ui/dialogs/dialogs.go`:
- Added a "Resources" section with two `widget.Entry` fields: CCADB URL and Refresh Days
- Validation mirrors `prefs.Load()` rules: empty URL resets to default; RefreshDays ≤ 0 resets to 30
- Values persist to `preferences.json` via `prefs.Save` on confirm

### CCADB Version Auto-Discovery

**2026-02-18** — Extended `EnsureCCADBCSV` in `internal/resources/fetcher.go`:
- Fetches `https://www.ccadb.org/resources` each refresh cycle and parses the HTML to
  extract the current CSV download URL via regex
- Auto-updates `prefs.Resources.CCADBURL` and `CachedFilename` when the URL changes;
  deletes stale cache file so the new version is fetched fresh
- Falls back gracefully to the stored URL if the resources page is unreachable
- CCADB CSV dialog shows Discovery URL, download URL, and cache file status
- Tests in `fetcher_test.go`: `TestDiscoverLatestCCADBURL` (mock HTML for v2/v3, no-match,
  404) and `TestEnsureCCADBCSV` (discovery update, version-change file deletion,
  discovery-failure fallback)

### PKCS#7 Support in AIA Downloads

**2026-02-18** — Extended `tryParseSingleCert()` in `internal/ui/chain/chain.go`:
- Detects PKCS#7/CMS via `Content-Type` response header or structure heuristic
- Extracts certificates from the bundle using `go.mozilla.org/pkcs7`
- Falls back gracefully with a clear error if parsing fails

## Phase 4 — Advanced Features (partial)

### Advanced Tab: Local-Only Certificate Count Color-Coding

**2026-02-18** — Updated `internal/ui/advanced/advanced.go`:
- "Certificates in Local Store Only" count uses `canvas.NewText` with `color.NRGBA`
- Green `{R:0, G:180, B:0, A:255}` when count is zero (local store fully covered by CCADB)
- Red `{R:200, G:0, B:0, A:255}` when one or more local-only certs are present

### Asynchronous Chain Building

**2026-02-19** — Updated `internal/ui/chain/chain.go`:
- `Build()` now accepts a `context.Context`; all HTTP fetches run in a background goroutine
- A spinner/status label shows "Building chain…" while fetching is in progress
- Opening a different certificate mid-build cancels the previous goroutine via context cancellation
- Partial chain displayed if any intermediate fetch fails (previously stopped at first error)

### PKCS#12 / PFX File Support

**2026-02-19** — Added `.pfx` / `.p12` to the file open dialog and drag-and-drop handler:
- Parses PKCS#12 bundles using `golang.org/x/crypto/pkcs12`
- Displays the end-entity certificate (private key material ignored entirely)
- If the bundle contains a chain it is displayed in the Chain tab without AIA fetching
- Password dialog (`dialogs.ShowPasswordPrompt`) prompts for passphrase on encrypted bundles;
  empty passphrase tried first to silently handle unencrypted files

### Certificate Signing Request (CSR) Viewing

**2026-02-19** — Added `.csr` / `.req` to the file open dialog and drag-and-drop handler:
- Parses CSRs using `crypto/x509.ParseCertificateRequest()`
- `summary.RenderCSR()` in `internal/ui/summary/summary_csr.go` renders Subject, SANs,
  public key, requested extensions, and signature into Summary + Details tabs
- "Type: Certificate Signing Request (not yet issued)" banner clearly distinguishes CSRs
  from issued certificates
- Chain tab displays "Not Applicable" placeholder
- `summary.ExportCSRText()` exports CSR details as plain text

## Phase 4 — Advanced Features (completed 2026-02-19)

### URL-Based Certificate Loading

**2026-02-19** — Added "Open URL…" to the File menu (`dialogs.ShowOpenURL`):
- Accepts a hostname or full HTTPS URL (with optional port)
- Dials via `certs.FetchTLSCerts` using `crypto/tls` in a background goroutine
- "Skip certificate verification" checkbox allows connecting to expired/self-signed servers
- Leaf cert loaded into viewer; Chain tab populated from the TLS handshake chain directly

### OCSP Status Checking

**2026-02-19** — Added "OCSP Status" row to the Summary tab when a certificate has an
OCSP URL in its AIA extension:
- "Check OCSP" button triggers an on-demand query via `certs.CheckOCSP` in a background
  goroutine (context-cancellable when a new cert is opened)
- Displays: Good / Revoked (reason + revocation time) / Unknown / Error
- Implemented in `internal/certs/ocsp_check.go` using `golang.org/x/crypto/ocsp`

### Validity Color-Coding

**2026-02-19** — Color-coded expiry dates throughout the UI:
- "Not After" / "Valid To" field shown in red if the certificate is expired
- Shown in yellow/amber if expiration is within the user-configurable warning threshold
  (default: 30 days, set via Preferences)
- Applied consistently in Summary tab and Chain sub-tabs
- Logic extracted to `internal/ui/summary/validity.go` (`ValidityColorName` pure function)
  with 7-case unit test in `validity_test.go`

### Recent Files

**2026-02-19** — Persists the last 10 opened file paths in `preferences.json`:
- `prefs.AddRecentFile()` pure function: dedup, prepend, cap at `MaxRecentFiles` (10)
- "Open Recent" submenu under File menu, built by the `rebuildMainMenu` closure
- Stale entries (file deleted) pruned automatically each time the menu is rebuilt
- Triggered on file dialog open, drag-and-drop, and "Open Recent" selection
- 6-case unit test in `prefs_test.go` covering: empty list, prepend, dedup, cap

### Certificate Details Export

**2026-02-19** — Two new File menu actions:
- "Export Details…" — `dialog.NewFileSave` writes a human-readable `.txt` file
- "Copy All to Clipboard" — calls `window.Clipboard().SetContent()` immediately
- Both disabled when no certificate or CSR is open; both re-created on every
  `rebuildMainMenu()` call so the disabled state always reflects current state
- `summary.ExportText()` and `summary.ExportCSRText()` in `internal/ui/summary/export.go`
  are pure functions (no Fyne dependency) mirroring `Render()` / `RenderCSR()` field-by-field
- Respects name style (OpenSSL/Windows) and hex separator preferences
- Tests: `TestExportText_ContainsKeyFields`, `TestExportText_WindowsNameStyle`,
  `TestExportCSRText_ContainsKeyFields` in `export_test.go`

### CRL Distribution Point Viewer

**2026-02-19** — Added "Fetch CRL" button to the Summary tab when a certificate has
CRL Distribution Points in its extensions:
- `certs.FetchCRL(ctx, url)` in `internal/certs/crl_fetch.go` — HTTP GET +
  `x509.ParseRevocationList`; pattern mirrors `fetchIssuerCert` in `ocsp_check.go`
- Button pattern exactly mirrors OCSP: background goroutine, `buttonCtx` capture,
  stale-cert guard via `buttonCtx.Err()`, disable/enable on fetch start/end
- Tries each Distribution Point URL in order, stops at first success
- Status label shows "N revoked" on success, "Error: …" on failure
- `dialogs.ShowCRL(win, rl, url)` in `internal/ui/dialogs/crl_dialog.go` opens a
  720×540 dialog with: header (issuer, this update, next update, entry count), a
  search `widget.Entry` for client-side filtering, and a `widget.List` for virtual
  rendering of potentially large entry lists
- `certs.FormatRevocationReason(code int) string` maps all 10 RFC 5280 reason codes
- Test: `TestFormatRevocationReason` (11 cases including unknown code) in `crl_fetch_test.go`

### Check CRL Button

**2026-02-19** — Added "Check CRL" button alongside "Fetch CRL" in the Summary tab:
- Fetches the CRL from the same Distribution Point URLs as "Fetch CRL"
- Calls `certs.CheckCertInCRL(cert, rl)` — linear scan of `RevokedCertificateEntries`
  by serial number; returns the matching entry or nil
- Result shown with `canvas.NewText` in the Summary grid:
  - **Good** in green `{R:0, G:180, B:0, A:255}` when serial not found in CRL
  - **REVOKED (reason) YYYY-MM-DD** in red `{R:200, G:0, B:0, A:255}` when found
- Context-cancellable: shares the CRL `buttonCtx` with the Fetch CRL button so
  opening a new cert cancels any in-flight CRL fetch

## Phase 4 — Advanced Features (completed 2026-02-21)

### Multiple Certificate Comparison

**2026-02-21** — Added a new Compare tab for side-by-side certificate diff:

New package `internal/ui/compare/`:
- `CompareLayout` — 3-column Fyne layout: field-name column auto-sized to max label
  width (same algorithm as `TightTwoColLayout`); remaining width split equally between
  cert A and cert B value columns
- `ExtractFields(cert, prefs) []Field` — extracts 23 named fields as plain strings:
  Common Name, Subject, Issuer, Serial, Not Before/After, Signature Algorithm,
  Public Key Algorithm + Size, Key Usage, Extended Key Usage, Basic Constraints,
  SKI, AKI, DNS/Email/IP/URI SANs, OCSP, CA Issuers, CRL Distribution Points,
  SHA-256 and SHA-1 fingerprints
- `BuildRows(certA, certB, prefs) []Row` — zips two `ExtractFields` results;
  sets `Differs=true` where values differ
- `Render(win, grid, certA, certB, prefs)` — clears the grid and populates
  3-column rows; differing rows use `theme.ColorNameWarning` on both sides
- 14 table-driven unit tests: field count, stable ordering, edge cases (empty SANs,
  absent BasicConstraints, CN fallback), diff detection

`main.go` additions:
- `currentCertB *x509.Certificate` + `currentFilenameA/B string` state
- `openCertB()` — file dialog restricted to cert files (no PKCS#12, no CSR)
- `renderCompareTab()` — rebuilds Compare tab content; shows prompt when no cert B
  is loaded; shows 3-column comparison when both certs are present
- `currentFilenameA` tracked across all cert-loading paths (file dialog, recent
  files, drag-and-drop, URL, PKCS#12)
- Cert B persists when cert A changes (compare against a fixed reference)
- Compare view refreshes automatically when preferences (e.g. hex separator) change
